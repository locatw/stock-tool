# Markdown Style Rules

IMPORTANT: Before writing or modifying Markdown files, you MUST read `doc/coding-guidelines/markdown/style.md` and follow all rules defined there.

Key rules:
- Do not use word decorations (`**word**` or `*word*`)
- Use hyphens or numbers for bullet points
- Keep sentences simple and concise
- One sentence per list item
- Exactly one empty line before and after headings (no more, no less)
- Exactly one space between hash symbols and heading text
- Exactly one empty line before and after lists (no more, no less)
