# Engineering Principles

These principles apply to all repository resources: code, documentation, configuration, and rule files alike.

## DRY (Don't Repeat Yourself)

Define each piece of knowledge in exactly one canonical location.
Other locations should depend on or reference the source rather than duplicating it.

## YAGNI (You Aren't Gonna Need It)

Do not add resources (code, documentation, configuration, rules) until actually needed by a current, concrete use case.
Remove resources when all consumers are gone.
