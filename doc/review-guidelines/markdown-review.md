# Markdown Review Guidelines

Apply these checks to every `.md` file touched by the change. Full rule definitions and examples are in `doc/coding-guidelines/markdown/style.md`.

## Checks

- No word decorations (`**word**` or `*word*`) — Warning
- Exactly one empty line before and after headings — Warning
- Exactly one space between hash symbols and heading text — Error
- Exactly one empty line before and after lists — Warning
- One sentence per list item — Warning
